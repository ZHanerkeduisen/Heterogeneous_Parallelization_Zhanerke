Задача 1. Введение в гетерогенную параллелизацию  (assignment2_part1.docx)
Тип: Теоретическое задание

Изучение принципов гетерогенной параллелизации (CPU + GPU).

Различие параллельных вычислений на CPU и GPU:

CPU — несколько мощных ядер, эффективен для последовательных и ветвящихся задач;

GPU — тысячи лёгких потоков, эффективен для массово параллельных операций.

Преимущества: ускорение вычислений за счёт использования сильных сторон разных устройств.

Примеры приложений:

Машинное обучение и нейронные сети (CPU: подготовка данных, GPU: обучение);

Научные и инженерные вычисления (симуляции, климатические модели);

Компьютерная графика и игры (GPU: рендеринг, CPU: логика);

Обработка изображений и видео (GPU: фильтры, декодирование видео);

HPC-системы (гибридные CPU+GPU суперкомпьютеры).

Вывод

Гетерогенная параллелизация позволяет значительно ускорить вычисления, распределяя задачи между CPU и GPU.

Задача 3. Параллельная сортировка с OpenMP  (assignment2task3.cpp)

Тип: Практическое задание

Цель: Освоить параллельную сортировку массивов на CPU с OpenMP.

Реализация

Алгоритм: сортировка выбором.

Реализованы:

- Последовательная версия;

- Параллельная версия (параллелизация внутреннего поиска минимального элемента).

- Массивы тестировались для 1000 и 10 000 элементов.

Внешний цикл не параллелится из-за зависимости итераций.

Задача 4. Сортировка на GPU с использованием CUDA  (assignment2task4.ipynb)

Тип: Практическое задание

Цель: Изучить реализацию алгоритмов сортировки на GPU и особенности гетерогенной параллелизации.

Реализация

Алгоритм: параллельная сортировка слиянием (Merge Sort).

Этапы:

Массив делится на подмассивы;

Каждый блок CUDA сортирует свой подмассив;

Выполняется многоступенчатое глобальное слияние.

Размеры массивов для тестирования: 10 000 и 100 000 элементов.

Замерено время выполнения.

Результаты

Массивы корректно сортируются.

Замеры времени показывают преимущество GPU при больших объёмах данных.

Вывод

CUDA позволяет эффективно распараллеливать алгоритмы с «разделяй и властвуй», но требует аккуратного управления памятью и синхронизацией потоков.
