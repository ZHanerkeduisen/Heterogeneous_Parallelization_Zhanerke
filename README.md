PRACTICE 2 

Описание 

В рамках практического задания реализованы три алгоритмы сортировки: Bubblr Sort (пузырьком), Selection Sort (выбором), и Insertion Sort (вставками). Это позволяет тестировать алгоритмы на массивах разного размера и измерять производитлеьность последовательных и параллельных версий с помощью OpenMP и библиотеки <chrono> (для измерение времени).

Цель: 
- Освоить параллельное программирование с использованием OpenMP на
CPU.
- Изучить и реализовать базовые алгоритмы сортировки (пузырьком,
выбором, вставкой) в параллельном варианте.
- Провести исследование производительности и сравнить параллельные и
последовательные реализации.

Задание 1 - Сортировки без параллелизма ()
- Реализовать последовательные версии Bubble Sort, Selection Sort, Insertion Sort.
- Замерить время выполнения алгоритмов на массивах размером 1000, 10000 и 100000 элементов.

Bubble Sort и Selection Sort - простые, наглядные алгоритмы со временем O(n^2)
Insertion Sort - последовательная версия, так как инерации взаимосвязаны и параллелизация невозможно без изменения логики алгоритма.

Задание 2 - Параллельная реализация с OpenMP

- Распараллелить внешниеи циклы Bubble Sort и Selection Sort с помошью директив OpenMP.
- Замерить время выполнения параллельных версий.
- Сравнить скорость работы последовательных и параллельных версий на массивах разных размеров.

Insertion Sort оставила последовательным, так как каждая итерация зависит от предыдущей - распараллеливание может привести к некоррекной сортировке.

Задание 3 - Сравнение производительности

- Для каждого алгоритма замеряеть время выполнения последовательной и параллельной версии.
- Результаты выводятся в консоль для наглядного сравнения.
- На больших массивах показывают существенное ускорение по сравнению с последовательным.
